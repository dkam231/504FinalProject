# Combined Workflow Design Document
## SUIM-to-FathomNet Foreground/Background Segmentation Pipeline

**Document status:** Approved implementation baseline  
**Primary audience:** project collaborators, reviewers, and future maintainers  
**Scope:** defines the architecture, data contracts, training strategy, and implementation rules for the final binary segmentation workflow

---

## 1. Executive Summary

This document defines the production-facing project structure for a binary underwater segmentation workflow built around two training stages:

1. **SUIM binary pretraining**
2. **FathomNet binary fine-tuning**

The workflow converts the original SUIM multiclass segmentation task into a foreground/background task and uses the resulting checkpoint as the initialization for FathomNet training. The objective is to create a single, coherent pipeline that is easier to understand, reproduce, and extend than the current repository’s split between a clean SUIM training stack and a separate FathomNet preprocessing stack.

The final implementation lives entirely inside `combined/` so collaborators can reason about the finished system without tracing legacy experimental files.

---

## 2. Problem Statement

The repository already contains two useful but only partially integrated components:

- a stable **SUIM U-Net training pipeline**
- a set of **FathomNet download and mask-preparation scripts**

However, the final project task is not multiclass SUIM segmentation. The actual target problem is **binary foreground/background segmentation** that transfers from SUIM to FathomNet. The new workflow therefore needs to:

- collapse SUIM labels into a binary task,
- preserve one consistent label convention across datasets,
- reuse the existing segmentation baseline instead of introducing unnecessary architectural drift,
- and provide clean train/evaluate entry points for both stages.

---

## 3. Project Objective

The `combined/` package is the single source of truth for the final system.

It must provide:

- binary mask generation rules for SUIM
- binary mask normalization rules for FathomNet
- a reusable baseline segmentation model
- a training entry point for SUIM pretraining
- a training entry point for FathomNet fine-tuning
- evaluation and prediction utilities
- a documented and reproducible folder structure

## 4. Dataset Decisions

## 4.1 SUIM

SUIM is originally a multiclass semantic segmentation dataset for underwater imagery. For this workflow, it is reduced to a binary task.

### Binary mapping
The following classes are treated as **background**:

- `BW` = background waterbody = RGB `(0, 0, 0)`
- `SR` = sand / sea-floor / rocks = RGB `(255, 255, 255)`

The following classes are treated as **foreground**:

- `HD` = human divers
- `PF` = plants / sea-grass
- `WR` = wrecks / ruins
- `RO` = robots / instruments
- `RI` = reefs / invertebrates
- `FV` = fish / vertebrates

### Resulting convention
- `0` → background
- `1` → foreground

This mapping is explicit and deterministic and must not be changed in hidden parts of the code.

## 4.2 FathomNet

For FathomNet, the workflow assumes images and masks are already available or have been generated by the repository’s existing preprocessing path.

The only requirement for the new package is that all FathomNet masks be normalized into the same binary convention:

- `0` → background
- `1` → foreground

Any non-zero or non-black mask content is interpreted as foreground unless the input format specifies a stricter convention.

---

## 5. Architecture Decisions

## 5.1 Baseline model

The workflow reuses the existing custom U-Net baseline already present in the repository. This is a deliberate decision.

### Rationale
- the model is already known to fit the project
- it is simple enough to debug
- it is sufficient for a strong baseline
- it minimizes engineering risk during transfer-learning experiments

The model is adapted to binary segmentation by using a **single output channel** rather than a multiclass segmentation head.

## 5.2 Output contract

The model output is a tensor of shape:

- `(B, 1, H, W)`

Targets are binary masks with shape:

- `(B, 1, H, W)`

Inference uses:

- `sigmoid(logits) > threshold`

Default threshold:
- `0.5`

---

## 6. Training Decisions

## 6.1 Stage A: SUIM pretraining

The model is initialized randomly and trained on the binary SUIM task.

Primary output:
- `suim_fg_bg_best.pth`

Checkpoint selection is based on validation segmentation quality, preferably foreground IoU or Dice.

## 6.2 Stage B: FathomNet fine-tuning

The model is initialized from the SUIM checkpoint and then fine-tuned on FathomNet.

Primary output:
- `fathomnet_fg_bg_best.pth`

This produces the final model used for the project’s FathomNet task.

---

## 7. Loss and Metric Decisions

### Default loss
- `BCEWithLogitsLoss`

### Optional improvement
- `BCE + Dice`

### Headline metrics
- pixel accuracy
- foreground IoU
- foreground Dice

Foreground IoU and Dice are the primary model-quality metrics.

---

## 8. Engineering Rules

1. **One binary convention everywhere**  
   `0 = background`, `1 = foreground`

2. **Final code stays inside `combined/`**  
   Legacy scripts may be consulted, but this folder is the authoritative implementation.

3. **Label conversion must be explicit**  
   There must be clearly named functions for SUIM multiclass-to-binary conversion and FathomNet mask normalization.

4. **Reproducibility is required**  
   Training scripts must expose data root, batch size, epochs, image size, learning rate, checkpoint paths, and seed.

5. **Visual sanity checks are part of the workflow**  
   Collaborators should be able to inspect image/mask pairs and binary conversion quality before training.

## 10. Final System Narrative

The final project workflow is:

> Convert SUIM from multiclass semantic segmentation into a binary foreground/background task, train a U-Net baseline on that task, then transfer the resulting checkpoint to FathomNet and fine-tune on binary FathomNet masks using the same label convention, model family, and training framework.
